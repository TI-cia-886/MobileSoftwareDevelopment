
# Lab5 Art Space 应用实验报告

## 一、应用展示内容
本应用为**经典油画艺术展示**应用，一共展示 **3 幅世界著名艺术作品**：
1. 星月夜 - 文森特·威廉·梵高（1889年）
2. 维纳河畔 - 巴勃罗·毕加索（1804年）
3. 拈花圣母 - 列奥纳多·达芬奇（1645年）

应用可以通过按钮切换作品，图片、作品名称、作者、创作年份会同步更新。

## 二、界面结构说明
整个界面按照实验要求划分为 **三大核心区块**，使用 Jetpack Compose 组件实现布局：

### 1. 艺术作品墙
- 组件：`Surface` + `Image`
- 作用：展示艺术作品图片
- 效果：添加了边框、阴影、内边距，实现画框效果
- 布局：占满屏幕宽度，固定高度，居中显示

### 2. 艺术作品说明区
- 组件：`Column` + `Text` + `Spacer`
- 作用：显示作品标题、艺术家、创作年份
- 样式：标题加粗放大，信息文字居中对齐
- 布局：垂直排列，间距均匀

### 3. 控制按钮区
- 组件：`Row` + `Button`
- 作用：提供 Previous / Next 切换按钮
- 布局：水平均匀排列，按钮宽度固定

### 整体嵌套结构
- 最外层：`Column`（垂直排列所有区块）
  - 上层：艺术作品墙
  - 中层：作品信息说明
  - 下层：切换控制按钮

## 三、Compose 状态管理实现
我使用 **Jetpack Compose 状态管理** 来控制当前显示的作品：
```kotlin
var currentArtwork by remember { mutableStateOf(1) }
```
- `currentArtwork`：状态变量，保存当前作品编号（1、2、3）
- `remember`：保证状态在界面重组时不会丢失
- `mutableStateOf`：让 Compose 可以监听变量变化，自动刷新界面

所有动态内容（图片、标题、作者、年份）都**根据这个状态变量自动更新**，没有写死固定数据。

## 四、Next / Previous 按钮逻辑
### 1. Next（下一张）按钮逻辑
- 如果当前是第 3 幅作品 → 切换回第 1 幅（循环）
- 其他情况 → 作品编号 +1
- 使用 `when` 表达式实现

### 2. Previous（上一张）按钮逻辑
- 如果当前是第 1 幅作品 → 切换到第 3 幅（循环）
- 其他情况 → 作品编号 -1
- 使用 `when` 表达式实现

两个按钮都实现了**循环切换**效果，符合实验要求。

## 五、遇到的问题与解决过程
1. **问题**：按钮点击后图片和文字不变化
   原因：没有使用 `mutableStateOf` 声明状态变量
   解决：使用 `remember { mutableStateOf(1) }` 让状态可被 Compose 监听

2. **问题**：图片没有画框效果
   解决：使用 `Surface` + `border` + `shadowElevation` 实现立体画框

3. **问题**：文字和按钮排版混乱、不居中
   解决：使用 `horizontalAlignment` 居中对齐，`Spacer` 控制间距

4. **问题**：作品信息显示错误
   解决：使用 `when` 表达式匹配作品编号，正确对应标题、作者、年份
